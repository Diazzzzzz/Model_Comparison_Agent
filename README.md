# 车型对比 Agent（MVP）

面向汽车销售场景：客户在几款同类车之间犹豫时，销售一键生成一份**扬长避短、正向引导**的对比 —— 站在我方车型立场，紧扣这位客户的在意点和顾虑。

一次生成三样东西：
1. **多维度对比表**（给销售看）
2. **一段话术**（销售照着说，30 秒抓住客户）
3. **可点击的场景化 H5**（发给客户，点车身部件弹卖点）

---

## 快速开始

```bash
pip install -r requirements.txt
python app.py
```

打开浏览器：**http://127.0.0.1:5000**

> 👉 **不配任何 API key 也能直接跑**：程序自动进入「演示模式」，用内置样例数据走通全流程，方便先看效果。

## 配置模型（可选，让它真用 AI 生成）

复制 `.env.example` 为 `.env`，填入 key：

- **`DEEPSEEK_API_KEY`** —— 文本模型，生成对比表 / 话术 / H5 文案。填了就走真模型。
- **`ARK_API_KEY`** —— 火山方舟「豆包·文生图 Seedream」，给 H5 生成场景背景图。不填则 H5 用内置矢量场景背景兜底。

## 目录结构

```
app.py                 Web 入口（Flask，路由 + 一键流程）
config.py              读 .env / 判断是否走演示模式
agent/
  comparison.py        核心编排：客户+两车 → 对比/话术/H5
  llm_client.py        DeepSeek 客户端（可换模型）
  image_client.py      火山 Seedream 客户端（可选）
  prompts.py           提示词 + H5 固定热区定义
  mock_data.py         演示模式的样例生成（用真实车参数拼）
data/
  car_library.py       车型库（模拟数据，⚠️可替换为真实库/RAG）
  customer_tags.py     客户标签 schema + 模拟客户（⚠️可替换为真实标签）
  mock_contexts.py     模拟"现场对话"（对应输入3，将来接工牌ASR）
templates/             index / result / h5 三个页面
```

## 说明 / 边界（MVP）

- **车型库、客户标签、对话上下文都是模拟数据**，且都做成了可替换模块——接你们真实数据时改对应文件即可，上层不动。
- **H5 里的车是通用 SUV 剪影**（占位用真图会失真）。真实产品换成**长城官方车图**，热区坐标不变。
- H5 场景背景：配了火山 key 就 AI 生成，否则用内置矢量场景。
- 结果暂存内存，重启即清空（MVP）。
