# 测试清单（车型对比 Agent MVP）

交接说明：本 MVP 用于验证"客户对比犹豫时，销售一键生成对比表 / 话术 / 客户 H5"这条链路。
下面清单照着点一遍，全绿即可。**无需任何 API key 即可完成 A–E**（演示模式）。

## 启动

```bash
pip install -r requirements.txt
python app.py          # 打开 http://127.0.0.1:5000
```

---

## A. 启动
- [ ] 服务能起来，浏览器打开 `http://127.0.0.1:5000` 正常显示界面
- [ ] 未配 key 时，右上角显示「演示模式 · 未配 DeepSeek key」

## B. 基本流程（演示模式，无需 key）
- [ ] 点「载入」示例客户 → 客户标签自动填满
- [ ] 意向车型 + 竞品两个下拉框**自动预选**为该客户的意向车/竞品
- [ ] 点「⚡ 一键生成」→ 出现：多维度对比表（含「✓ 占优」标记）、话术、客户 H5 入口
- [ ] 点「打开客户 H5」→ 车身上 5 个红点，逐个点击弹出对应卖点

## C. 现场对话自动提炼
- [ ] 选一段对话 → 点「🎙️ 提炼标签」→ 上方以气泡展示该段对话（销售/客户左右分开）
- [ ] 下方客户标签与该段对话内容一致（选李女士那段出李女士，不串号）
- [ ] 意向车 / 竞品下拉随之预选

## D. 关键回归：字段重名 bug（曾导致「车型没找到」）
- [ ] 载入客户后，手动把「正在纠结的竞品」文本框改成任意乱写内容
- [ ] 下拉框正常选车 → 点生成 → **正常出结果，不报「车型没找到」**

## E. 边界 / 防呆
- [ ] 我方车与竞品选同一款 → 提示「不能是同一款」，不崩溃（返回 400 提示页）
- [ ] 选了不存在的车型（异常提交）→ 返回 400 友好提示，不崩 500
- [ ] 客户标签全部留空 → 仍能生成，不报错

## F. 接入真实模型（配置 key 后）
- [ ] `.env` 填 `DEEPSEEK_API_KEY` 重启 → 结果页顶部标签变为「DeepSeek 生成(deepseek-chat)」，对比/话术为真实 AI 生成
- [ ] `.env` 填火山 `ARK_API_KEY` + 正确 `ARK_IMAGE_MODEL` 重启 → H5 背景为 AI 生成的场景图
- [ ] 未配火山 key 时，H5 背景为内置渐变背景（预期行为，非 bug）

---

## 已知边界（MVP，非缺陷）
- 车型库 / 客户标签 / 对话上下文均为**模拟数据**，均为可替换模块（见 `data/`）。
- H5 中的车为**通用 SUV 剪影占位**，待替换为长城官方车图（热区坐标不变）。
- 生成结果存内存，重启即清空。
