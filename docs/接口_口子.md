# 车型对比 Agent · 对外接口（给 Sales Agent 的「口子」）

Sales Agent 的「对比竞品」节点通过这个接口调用车型对比能力。
两种调法，返回结构**完全一致**，按部署方式二选一。

## 调法一：进程内直接调用（同一 Python 服务时最省事）

```python
from agent.comparison import generate_comparison

result = generate_comparison(customer, our_car, rival_car)
```

- `customer`: dict，客户标签（`name/budget/usage/care_most/worry/...` 都可缺省）
- `our_car` / `rival_car`: str，必须是车库中的车名
- 返回：见下方「返回结构」

## 调法二：HTTP JSON API（Sales Agent 独立部署 / 跨服务时）

```
POST /api/compare
Content-Type: application/json

{
  "customer": {"name":"王先生","budget":"15万内","care_most":"安全、空间","worry":"不保值"},
  "our_car": "长城哈弗H6",
  "rival_car": "丰田RAV4荣放"
}
```

- 成功：`200` + 结果 JSON（见下）
- 车型无效：`400` + `{error, valid_our[], valid_rival[]}`
- 内部错误：`500` + `{error}`

## 返回结构（两种调法一致）

```json
{
  "summary": "一句话总结",
  "dimensions": [
    {"name":"安全","our_value":"...","rival_value":"...","winner":"ours|rival|tie","comment":"..."}
  ],
  "talk_track": "给销售照着说的话术",
  "h5": {"scene_title":"...","hotspots":[...],"scene_image_url":null},
  "our_car": { }, "rival_car": { }, "customer": { },
  "_mode": "演示模式(...) | DeepSeek 生成(...)",
  "_warn": "（可选）数字防幻觉告警"
}
```

## Sales Agent 侧怎么接

在 Sales Agent 仓库的 `tools/model_comparison.py` 里，把现在的**工具桩**换成调用本接口即可 ——
Sales Agent 的引擎 / 节点 / 状态**都不用改**：

```python
# tools/model_comparison.py（接入后）
import requests

def model_comparison(customer, our_car, rival_car):
    resp = requests.post("http://<车型对比服务>/api/compare",
                         json={"customer": customer, "our_car": our_car, "rival_car": rival_car})
    return resp.json()
```

> 现状：`/api/compare` 口子已就绪，属「先留口子」阶段。车型对比 Agent 保持为**干净的小工具**
> （客户 + 两款车 → 对比表 / 话术 / H5），对话式演示由 **Sales Agent** 负责，不放在这里。
> 真正接进 Sales Agent 是后续步骤（等车型对比本身的功能改动定稿后再做）。
