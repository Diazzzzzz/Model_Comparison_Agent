# 车型对比 Agent · 车企数据需求清单

> 目的：把这个 MVP 接到长城真实数据上线所需的**全部数据/素材/授权**一次列清。
> 优先级：**P0 = 上线必需，先给**；**P1 = 首发后补齐即可**。
> 每一项都标了「给什么」「什么格式」「谁提供」，方便对口到长城相应部门。

---

## 一、车型参数库（P0 · 最核心）

对比表的每个数字都从这里取、并锁定，AI 不允许改动或编造 —— **所以这份数据的准确性 = 产品可信度**。

**需要：我方车型 + 主要竞品**，且**按市场/国家分行**（价格、月供、配置各国不同）。

| 字段 | 说明 | 示例 |
|---|---|---|
| 车型名 / 品牌 / 车级 | 唯一标识 | 哈弗H6 / 长城 / 紧凑型SUV |
| 是否我方 | 我方 or 竞品 | 是 |
| 指导价（分市场/币种） | 当地货币 | ZAR 399,900 |
| 金融方案 | 首付比例 / 期数 / 利率 / 参考月供 | 首付30% / 48期 / 参考月供 ZAR xxx |
| 动力 | 排量/马力/变速箱 | 2.0T 204hp 8AT |
| 能耗/油耗（或续航） | 综合油耗；纯电给续航 | 7.2L/100km |
| 轴距 / 后备箱 | 空间数据 | 2738mm / 560L |
| 底盘/悬架 | | 前麦弗逊后多连杆 |
| 安全 | 气囊/主动安全/碰撞星级 | 6气囊+L2 / NCAP 5星 |
| 车身用钢 | 高强钢占比等 | 高强度钢78% |
| 智能座舱 | 屏幕/车机/语言支持 | 14.6"，支持英语/当地语 |
| 质保政策（分市场） | 整车/三电/年限公里 | 5年/15万公里 |
| 保养成本 | 小保养价/周期/套餐 | 当地货币 xxx |
| 售后/服务网络 | 网点数/覆盖/救援 | 当地 xx 家，含道路救援 |

- **格式**：Excel / CSV / 接口均可，一车一行（含市场维度），字段名固定。
- **提供方**：长城产品/数据部门（竞品数据见第五节，注意合规）。
- **更新机制**：需明确**谁、多久更新一次**（价格/促销变动频繁）。

> 对应代码：`data/car_library.py`（替换后 `get_car()` 返回同结构即可，上层不动）。

---

## 二、价格与金融（P0 · 按市场）

- 各首发市场的**指导价、促销/优惠、金融方案**（首付、期数、利率、月供）、**币种**。
- 单列，因为它**变动最快**、且**每个国家不一样**，最好能接口对接而非静态表。

---

## 三、售后 / 服务网络数据（P0 · 出海关键）

海外客户最大的顾虑是"坏了谁修"。需要：
- 各市场**服务网点清单 / 覆盖范围**；
- **保养菜单价**（常见项目工时+配件）；
- **质保政策**、道路救援政策。
- 提供方：长城海外售后/网络部门。

---

## 四、车图素材（P0 · H5 用）

H5 现在是通用 SUV 剪影，需换成长城官方车图 + 标热区。

- **每车多种车身颜色的官方图**，**同一拍摄角度**（3/4 侧前最佳）；
- 建议**透明底 PNG**（系统也能自动抠图，但原图干净更好）；长边 ≥ 1600px；
- 颜色名（曜石黑/星河银…）、拼色车型标注双色。
- 提供方：长城市场/设计部门。
- 交付后由内部工具 `/admin` 标 5 个热区(车头/轮胎/车身/车顶/座舱)，H5 复用同一套坐标。

---

## 五、卖点 / 话术 / 竞品对标素材（P0~P1）

这块决定"说得好不好"，也是找补库/异议库的种子。

- **官方核心卖点**、产品培训资料（P0，种子话术）；
- **常见异议 Top10 + 标准应对**（P0）→ 直接变成"找补话术库"；
- **客户最常问的对比点 Top10**（P0）→ 校准对比维度；
- **竞品对标数据**（P1）：⚠️**必须是长城官方授权、可对外使用**的版本，并注明**来源与日期**（对外对比广告受当地法规约束，见第八节）。
- 提供方：长城产品培训 / 市场部。

---

## 六、客户标签 / CDP 对接（P1）

现在是模拟客户，正式版由系统自动带入。需要：
- 客户标签**字段 schema**（意向车型、预算、用途、家庭、在意点、顾虑、意向热度、竞品…）；
- **对接接口/权限**（从哪个客户库/CDP 取）；
- **意向分级定义**（H/W/C 级怎么界定）。
- 对应代码：`data/customer_tags.py`。

---

## 七、现场对话 / 工牌 ASR（P1）

对应"从对话自动提炼客户标签"。需要：
- **工牌/录音硬件**方案与**音频→文本(ASR)接口**；
- 支持的**语言**（英语/当地语）；
- 隐私合规（录音授权）。
- 对应代码：`data/mock_contexts.py`（现在是模拟对话）。

---

## 八、多语言 / 本地化 / 合规（P0 决策 · 出海必答）

⚠️ 这几条现在代码里没有，但**出海必须先定**：

- **输出语言**：英语？当地语？中英对照？（现版本全中文）
- **术语表 / 品牌用语规范**：车型名、配置、卖点的官方译名。
- **合规红线**：
  - 对外**对比广告**在南非/欧盟等地受监管——对竞品的每条说法必须**真实、可举证**；
  - 需明确**竞品数据的授权范围**和**对外话术的审批流程**（谁审、审什么）；
  - 数据合规（如南非 POPIA）：客户数据存储/跨境的要求。
- 提供方：长城法务 / 品牌 / 合规。

## 优先级速览

| 先给（P0） | 后补（P1） |
|---|---|
| 车型参数库（含竞品，分市场） | 客户标签/CDP 对接 |
| 价格与金融（分市场） | 工牌 ASR / 现场对话 |
| 售后/服务网络数据 | 竞品对标深度数据 |
| 官方车图素材 | — |
| 核心卖点 + 异议Top10 + 对比点Top10 | — |
| 语言/合规红线（决策项） | — |

> 建议：**先要"一个首发市场（南非）+ 我方主推1~2款 + 对标竞品"的完整数据跑通闭环**，再横向铺其他市场/车型。
